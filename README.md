## 何をしているか

ヘッドレスブラウザ（selenium)でWebサイトの要素を取得し、その内容の文字列をコンソールに出力するだけ。

## 気を付けたこと

### 一切ヘッドレスChromeを画面上に表示させないで要素情報を取得すること

selenumのheadless modeを使っただけでは、どうしてもChromeのGUIがひょっこり表示されてしまっていた。

そこで、いろいろ試した結果、seleniumで取得した全体のページのソースコードをlxmlにパスして、

それからXPathを使って文章を抽出するようにしてみた。

### 一応user-agentで「自分はbotです」と通知すること

``changedetection.io``をVPSや個人サーバーで運用していた時にも感じていた問題点。

「スクレイピングは無制限に許可されているケースは少ない」という点。

この点、「常識の範囲内なら」とか、「よほど負荷をかけて迷惑をかけなければ黙認では」とか、

いろいろ立場はあると思う。

しかし、Webがある程度の「性善説」で自己統治するコミュニティの面があることを考えると、

ある程度の「誠実さ」をもって技術を使用すべきだと思う。

今回、そのような立場から、user-agentを``MyBot/1.0``として、サーバー側に通知するようにしてみた。
