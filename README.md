## 何をしているか

ヘッドレスブラウザ（selenium)でWebサイトのJavaScript実行後のHTMLソースを取得し、lxmlに投げ、

[スクレイピングOKなこのサイト様](https://quotes.toscrape.com/)

の、

歴史的に有名な偉人の名言

の要素をまず出力し、

次に、XPathで絞り込み、

該当する名言自体の文字列

を取得してコンソールに表示する。

## 気を付けたこと

### 要JavaScriptサイトにも対応

今回使わせていただいたサイトは、seleniumでソース取得しなくても名言部分の要素は取得可能です。

しかし、このようなシンプルなサイトは実際のサイトでは少なく、ほとんどがJSの実行を前提としたものですので、

テストスクリプトでありながら最低限の実用性の確保をする(要JSサイトでも初回ロード時の完全なHTMLソースを取得できる)ように``selenium``を使いました。

### 一切ヘッドレスChromeを画面上に表示させないでJS実行済みソースを取得すること

selenumのheadless modeを使っただけでは、どうしてもChromeのGUIがひょっこり表示されてしまっていた。

当初、Windowsのスタートアップに入れて、起動時だけお気に入りのサイトの差分チェックをできたら面白いな([Distill](distill.io)などの二番煎じですが)と思っておりました。

そこで、いろいろ試した結果、seleniumで取得した全体のページのソースコードをlxmlにパスして、

それからXPathを使って文章を抽出するようにしてみた。

### 一応user-agentで「自分はbotです」と通知すること

``changedetection.io``をVPSや個人サーバーで運用していた時にも感じていた問題点。

「スクレイピングは無制限に許可されているケースは少ない」という点。

この点、「常識の範囲内なら」とか、「よほど負荷をかけて迷惑をかけなければ黙認では」とか、

いろいろ立場はあると思う。

しかし、Webがある程度の「性善説」で自己統治するコミュニティの面があることを考えると、

ある程度の「誠実さ」をもって技術を使用すべきだと思う。

今回、そのような立場から、user-agentを``MyBot/1.0``として、サーバー側に通知するようにしてみた。
